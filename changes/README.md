# Changes

`changes/` is for active multi-step work only.

- Use `changes/<issue-id>-slug/` for standard and structural changes.
- Do not create a change pack for tiny fixes.
- Keep change packs short and focused on the current implementation.
- Promote durable knowledge before merge.
- Remove the change pack from the default branch after merge.

See [`docs/runbooks/change-lifecycle.md`](../docs/runbooks/change-lifecycle.md) for the full workflow.
